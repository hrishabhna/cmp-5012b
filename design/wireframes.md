## Screen 1,2,3 

![System Wireframes](wireframes.png)

The low-fidelity wireframes above represents

* **Screen 1: Main Dashboard**
    * **Goal:** To provide the user with an immediate overview of their daily progress.
    * **Key Elements:** A central progress ring for step counts and specific data cards for secondary metrics (Calories and Water).
    * **Interaction:** The navigation bar at the bottom ensures a "flat" hierarchy, allowing the user to switch between logging and social features with a single tap.

* **Screen 2: Log Activity Form**
    * **Goal:** To capture exercise data quickly and accurately.
    * **System State:** This screen demonstrates the **Success State**. Once the user clicks "Save Activity," a green feedback message appears to confirm the data has been successfully written to the JSON store.
    * **Components:** Dropdown menus and numeric input fields are used to ensure data integrity and prevent user error.

* **Screen 3: Group Leaderboard**
    * **Goal:** To facilitate social competition, a key requirement for the insurance-linked health scheme.

    * **Call to Action:** A prominent button at the bottom allows for easy group expansion through invite codes, supporting the "Social and Competitive" system responsibility.
